com.apama.scenario
Event ScenarioServiceUpdaterMultipleInstances

Utility event for tracking configuration for a given scenario. This event is suitable for use by monitors which spawn per scenario, but not per instance. (e.g. ScenarioService, DataViewService)

actions starting with an underscore should be considered private and not called by users of this event.

Note that this updater only honours sendThrottledUser changes at the next throttling period (it does not record the owner if sendThrottledUser is not true)
